## What changed

- **New dependencies**: `@capacitor/filesystem`, `@capacitor/share`, `@capawesome/capacitor-file-picker`, pinned to major version `8` to match game-shelf and the rest of the Capacitor stack.
- **New**: `src/app/core/utils/native-platform.util.ts` — verbatim port of game-shelf's `Capacitor.isNativePlatform()`/`getPlatform()` wrappers.
- **New**: `src/app/core/utils/share-file.util.ts` — verbatim port of game-shelf's `presentShareFile()`. Web path tries `navigator.share`/`canShare` with a constructed `File`, falling back to the classic Blob + anchor-download trick; native path stages the file via `Filesystem.writeFile()` into `Directory.Cache`, opens `Share.share()`, and deletes the cached file in a `finally`. Cancels (`AbortError` or an "abort|cancel" message) are treated as normal, not errors.
- **New**: `src/app/core/utils/pick-file.util.ts` — not a verbatim port (game-shelf's `pickCsvTextFile()`/image-pickers are CSV/image-specific; go-gather only ever imports one JSON bundle). Follows the same shape: web falls back to a programmatically-created hidden `<input type=file accept=".json,application/json">` with a window-focus-based 500ms cancel-detection fallback (browsers don't universally fire a `cancel` event on `<input>`); native uses `FilePicker.pickFiles({ types: ['application/json', 'text/plain'], limit: 1 })`, converting the picked result into a `File` via its `.blob` directly or, when only a `path` is given, `fetch(Capacitor.convertFileSrc(picked.path))`.
- **`gather.page.ts`/`.html`**: restored the Export/Import `ion-button`s to the header toolbar (dropped in Group 1 rather than stubbed with a throwaway web-only implementation). `exportBundle()` calls `presentShareFile()` with `JSON.stringify(userDataService.exportBundle(), null, 2)` and a `go-gather-backup-${timestamp}.json` filename. `triggerImport()` calls `pickJsonTextFile()`, then `parseExportBundle()` (ported verbatim from go-gather-next's `AppComponent` — accepts either a full `ExportBundle` or a legacy bare progress-array, defaulting missing fields via `??`, throwing `'Unrecognized backup file format'` otherwise), then `userDataService.importBundle(bundle).subscribe(...)`.
- **Added toast feedback on import** (success and failure) via `ToastController` — the go-gather-next original only `console.error`'d on failure with no success feedback at all. This is a deliberate small UX improvement, not scope creep: silent success is a bad pattern on a phone with no dev console to check, and it matches the toast pattern already established for copy-to-clipboard in an earlier group. Export failures are treated as cancels by `presentShareFile` already and don't get a toast; genuine unexpected export errors still bubble up uncaught.

## `PrivacyInfo.xcprivacy`

Added `ios/App/App/PrivacyInfo.xcprivacy`, copied verbatim from game-shelf's file, which declares **two** Required Reason API categories:

- `NSPrivacyAccessedAPICategoryFileTimestamp`, reason `C617.1` — required because `@capacitor/filesystem` (newly added this task) touches file timestamps under the hood.
- `NSPrivacyAccessedAPICategoryUserDefaults`, reason `CA92.1` — required because `@capacitor/preferences` reads/writes `UserDefaults`. This dependency was already in go-gather from Phase 2's `PreferenceStorageService`, but had no `ios/` project to declare it in until now — so this task is the first opportunity to add it, even though it's nominally about Filesystem/Share/FilePicker.

Registering the file so it actually bundles into the app required four manual edits to `ios/App/App.xcodeproj/project.pbxproj` (go-gather's project is single-target, unlike game-shelf's dual-target project which needs duplicate entries): one `PBXFileReference`, one `PBXBuildFile` referencing it, one addition to the `App` group's `children` array, and one addition to the `Resources` build phase's `files` array — each keyed by a freshly-generated, collision-checked 24-hex-char object ID. Validated with `plutil -lint` (returned "OK") before running `npx cap sync ios`, which completed successfully and reported "Found 9 Capacitor plugins for ios" including all three newly added ones (via Swift Package Manager, not CocoaPods).
